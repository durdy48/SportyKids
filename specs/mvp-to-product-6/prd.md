# Phase 6, PRD 1: Subscription Monetization with RevenueCat

| Field | Value |
|-------|-------|
| **Phase** | 6.1 — Subscription Monetization |
| **Priority** | P0 (revenue enables all future development) |
| **Target** | Month 1 post-launch (v1.1.0) |

---

## 1. Overview / Problem Statement

SportyKids is live on both app stores with zero revenue. Without monetization, development is funded entirely out of pocket, giving the product a finite runway. The app needs a sustainable revenue model that:

- Does not compromise the child-safe experience
- Lets parents (not children) make purchasing decisions
- Works across iOS, Android, and eventually web
- Has minimal implementation overhead for a solo developer

A freemium subscription model via **RevenueCat** addresses all four points. RevenueCat abstracts App Store / Google Play billing, handles receipt validation, renewal, cancellation, and provides a webhook system for server-side entitlement checks. It is free until $2,500 MRR.

---

## 2. Goals and Non-Goals

### Goals

1. **Two-tier model**: Free (limited) and Premium ($2.99/mo or $24.99/yr) enforced server-side.
2. **Family plan**: One parent subscription covers up to 3 linked children via `parentUserId`.
3. **RevenueCat integration**: Mobile SDK for purchasing, server webhook for entitlement sync.
4. **Limit enforcement**: Free tier limits enforced at the API level, not just in UI.
5. **Paywall UX**: Contextual upgrade prompts when limits are hit + dedicated upgrade screen.
6. **Anonymous upgrade path**: Anonymous users must register (email/OAuth) before purchasing.
7. **Restore purchases**: Users can restore purchases on new devices or reinstalls.

### Non-Goals

- Web purchases via Stripe (future PRD — web shows "download app to subscribe")
- Free trial period (defer until churn data is available)
- Promotional offers or coupon codes
- Multiple premium tiers (e.g., a "Pro" tier)
- Ad-supported free tier
- Offline purchase validation (require network for purchase)
- In-app purchase of individual items (stickers, etc.)

## 3. Tier Definition

| Feature | Free | Premium ($2.99/mo or $24.99/yr) |
|---------|------|----------------------------------|
| News articles per day | 5 | Unlimited |
| Quiz questions per session | 3 | Unlimited |
| Reels per day | 5 | Unlimited |
| Sports | 1 (first `favoriteSports` entry) | All 8 |
| Weekly digest PDF | No | Yes |
| Sticker collection | Basic drops only | Premium rarity drops |
| Parental time limits | Yes | Yes |
| Schedule lock | Yes | Yes |
| Ad-free | Yes (always) | Yes (always) |

**Important**: Parental controls (time limits, schedule lock, format restrictions) always take precedence over subscription tier. A premium user whose parent has set a 30-minute daily limit is still capped at 30 minutes.

## 5. API Endpoints

### 5.1 `GET /api/subscription/status/:userId`

Returns the current subscription status for a user, including remaining daily limits.

**Auth**: `requireAuth` (caller must be the user or their parent).

**Response** (200):

```json
{
  "tier": "free",
  "expiry": null,
  "limits": {
    "newsPerDay": 5,
    "reelsPerDay": 5,
    "quizQuestionsPerSession": 3,
    "sportsAllowed": ["football"]
  },
  "usage": {
    "newsToday": 3,
    "reelsToday": 1,
    "quizToday": 0
  },
  "canUpgrade": true,
  "familyPlan": false,
  "childCount": 0
}
```

**Logic**:

1. Fetch user with `subscriptionTier`, `subscriptionExpiry`, `favoriteSports`, `parentUserId`.
2. If user is a child with `parentUserId`, check the parent's subscription tier (family plan propagation).
3. Count today's `ActivityLog` entries by type (`news_viewed`, `reels_viewed`, `quizzes_played`).
4. If tier is `premium` and `subscriptionExpiry` is in the past, treat as `free` (grace period handling done by RevenueCat webhook, this is a safety net).
5. For free tier, `sportsAllowed` = `[favoriteSports[0]]` (first sport only). If `favoriteSports` is empty, allow all (they haven't onboarded yet).

### 5.2 `POST /api/subscription/webhook`

RevenueCat server-to-server webhook for subscription lifecycle events.

**Auth**: Shared secret via `Authorization: Bearer <REVENUECAT_WEBHOOK_SECRET>` header.

**Request body** (RevenueCat webhook format):

```json
{
  "api_version": "1.0",
  "event": {
    "type": "INITIAL_PURCHASE",
    "app_user_id": "<parentalProfile.revenuecatCustomerId>",
    "product_id": "sportykids_premium_monthly",
    "expiration_at_ms": 1735689600000,
    "environment": "PRODUCTION"
  }
}
```

**Supported event types**:

| Event | Action |
|-------|--------|
| `INITIAL_PURCHASE` | Set parent + all children to `subscriptionTier='premium'`, set `subscriptionExpiry` |
| `RENEWAL` | Update `subscriptionExpiry` for parent + children |
| `CANCELLATION` | Keep `premium` until `subscriptionExpiry`, then reverts to `free` on next status check |
| `BILLING_ISSUE` | Log warning, keep `premium` (RevenueCat retries billing) |
| `EXPIRATION` | Set parent + all children to `subscriptionTier='free'`, clear `subscriptionExpiry` |
| `PRODUCT_CHANGE` | Update `subscriptionExpiry` based on new product |

**Response**: 200 OK (empty body). RevenueCat retries on non-2xx.

**Logic**:

1. Validate `Authorization` header against `REVENUECAT_WEBHOOK_SECRET` env var.
2. Find `ParentalProfile` by `revenuecatCustomerId` matching `event.app_user_id`.
3. Find parent `User` via profile.
4. Find all children via `User.where({ parentUserId: parent.id })`.
5. Update `subscriptionTier` and `subscriptionExpiry` on parent + all children in a transaction.
6. Log event via Pino.

### 5.3 Modified Existing Endpoints — Limit Enforcement

#### `GET /api/news` — Free tier: 5 reads/day

Add limit checking **after** the existing `parentalGuard` middleware:

1. If user has `subscriptionTier='free'`, count today's `ActivityLog` where `type='news_viewed'` and `userId` matches.
2. If count >= 5, return 403 with `error: 'subscription_limit_reached'` and `limitType: 'news'`.
3. For free tier, if `sport` query param is set and it's not the user's first `favoriteSport`, return 403 with `error: 'subscription_sport_restricted'`.

#### `GET /api/reels` — Free tier: 5 views/day

Same pattern: count `reels_viewed` ActivityLog entries. If >= 5, return 403 with `subscription_limit_reached`.

#### `GET /api/quiz/questions` — Free tier: 3 questions/session

The `count` query parameter already controls how many questions are returned. For free tier:
1. Override `count` to `Math.min(requestedCount, 3)`.
2. If user has already answered 3+ questions today (via `ActivityLog` type `quizzes_played`), return 403.

### 5.4 Subscription Limit Middleware

New file: `apps/api/src/middleware/subscription-guard.ts`

```typescript
// Pseudocode — actual implementation will follow this pattern
export function subscriptionGuard(format: 'news' | 'reels' | 'quiz') {
  return async (req, res, next) => {
    const userId = req.query.userId || req.headers['x-user-id'];
    if (!userId) return next();

    const user = await getUser(userId);  // cached
    const tier = resolveEffectiveTier(user);  // checks parent subscription for children

    if (tier === 'premium') return next();

    // Free tier checks
    const todayCount = await countTodayActivity(userId, format);
    const limit = FREE_TIER_LIMITS[`${format}PerDay`];

    if (todayCount >= limit) {
      throw new AuthorizationError('Daily limit reached', {
        error: 'subscription_limit_reached',
        limitType: format,
        limit,
        used: todayCount,
        tier: 'free',
      });
    }

    // Sport restriction for free tier
    if (format === 'news' || format === 'reels') {
      const sport = req.query.sport;
      const allowedSport = user.favoriteSports[0];
      if (sport && allowedSport && sport !== allowedSport) {
        throw new AuthorizationError('Sport not available on free plan', {
          error: 'subscription_sport_restricted',
          allowedSports: [allowedSport],
          tier: 'free',
        });
      }
    }

    next();
  };
}
```

**Middleware order on content routes**: `authMiddleware` -> `parentalGuard` -> `subscriptionGuard` -> route handler.

Parental limits always take precedence: if a parent has set a 30-minute time limit and the child has hit it, the `parentalGuard` returns 403 before `subscriptionGuard` runs. The subscription limit is a separate daily count on top of parental restrictions.

### 5.5 Tier Resolution for Children (Family Plan)

```typescript
async function resolveEffectiveTier(user: User): Promise<SubscriptionTier> {
  // If user is premium directly, return premium
  if (user.subscriptionTier === 'premium' && (!user.subscriptionExpiry || user.subscriptionExpiry > new Date())) {
    return 'premium';
  }

  // If user is a child with a parent, check parent's tier
  if (user.parentUserId) {
    const parent = await prisma.user.findUnique({
      where: { id: user.parentUserId },
      select: { subscriptionTier: true, subscriptionExpiry: true },
    });
    if (parent?.subscriptionTier === 'premium' && (!parent.subscriptionExpiry || parent.subscriptionExpiry > new Date())) {
      return 'premium';
    }
  }

  return 'free';
}
```

## 6. RevenueCat Setup

### 6.1 Account & Project

1. Create account at [app.revenuecat.com](https://app.revenuecat.com).
2. Create project "SportyKids".
3. Add two platforms: **Apple App Store** and **Google Play Store**.
4. For Apple: upload App Store Connect Shared Secret (from App Store Connect > General > App Information > App-Specific Shared Secret).
5. For Google: upload Google Play Service Account JSON key (from Google Cloud Console > IAM > Service Accounts > create key for Play Developer API).

### 6.2 Products Configuration

Create products in App Store Connect and Google Play Console first:

| Product ID | Platform | Type | Price |
|-----------|----------|------|-------|
| `sportykids_premium_monthly` | Both | Auto-renewable subscription | $2.99/mo (EUR equivalent) |
| `sportykids_premium_yearly` | Both | Auto-renewable subscription | $24.99/yr (EUR equivalent) |

In RevenueCat:
1. Create **Entitlement** named `premium`.
2. Create **Offering** named `default`.
3. Create two **Packages** under the offering: `$rc_monthly` and `$rc_annual`.
4. Map the store products to the packages.
5. Assign the `premium` entitlement to both products.

### 6.3 Webhook

1. In RevenueCat dashboard > Project Settings > Webhooks, add a new webhook.
2. URL: `https://sportykids-api.fly.dev/api/subscription/webhook`
3. Authorization header: `Bearer <REVENUECAT_WEBHOOK_SECRET>`
4. Events to send: `INITIAL_PURCHASE`, `RENEWAL`, `CANCELLATION`, `BILLING_ISSUE`, `EXPIRATION`, `PRODUCT_CHANGE`.

### 6.4 Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `REVENUECAT_API_KEY_APPLE` | Yes (mobile) | RevenueCat Apple API key (public, embedded in mobile app) |
| `REVENUECAT_API_KEY_GOOGLE` | Yes (mobile) | RevenueCat Google API key (public, embedded in mobile app) |
| `REVENUECAT_WEBHOOK_SECRET` | Yes (API) | Shared secret for webhook authentication |
| `REVENUECAT_API_SECRET` | No | RevenueCat secret API key (for server-side API calls, if needed) |

```typescript
import Purchases, { LOG_LEVEL } from 'react-native-purchases';
import { Platform } from 'react-native';
import Constants from 'expo-constants';

const API_KEY = Platform.OS === 'ios'
  ? Constants.expoConfig?.extra?.REVENUECAT_API_KEY_APPLE
  : Constants.expoConfig?.extra?.REVENUECAT_API_KEY_GOOGLE;

export async function initPurchases(userId: string): Promise<void> {
  if (__DEV__) {
    Purchases.setLogLevel(LOG_LEVEL.DEBUG);

  await Purchases.configure({ apiKey: API_KEY, appUserID: userId });

export async function getOfferings() {
  const offerings = await Purchases.getOfferings();
  return offerings.current;
}

export async function purchasePackage(pkg: any) {
  const result = await Purchases.purchasePackage(pkg);
  return result.customerInfo;
}

export async function restorePurchases() {
  const customerInfo = await Purchases.restorePurchases();
  return customerInfo;
}

export async function getCustomerInfo() {
  const info = await Purchases.getCustomerInfo();
  return info;
}
```

## 13. Acceptance Criteria

### AC-1: Data Model

- [ ] `subscriptionTier` field exists on User model with default `'free'`.
- [ ] `subscriptionExpiry` field exists on User model as nullable DateTime.
- [ ] `revenuecatCustomerId` field exists on ParentalProfile with unique constraint.
- [ ] Migration runs cleanly on existing data (all existing users become `'free'`).

### AC-2: Subscription Status Endpoint

- [ ] `GET /api/subscription/status/:userId` returns correct tier, limits, and usage.
- [ ] Children inherit premium status from their parent (via `parentUserId`).
- [ ] Usage counts reset at UTC midnight (not user timezone).
- [ ] Expired subscriptions are treated as free tier.
- [ ] Requires authentication; returns 401 for unauthenticated requests.
- [ ] A parent can query the status of their child; a child can only query their own.

### AC-3: Webhook

- [ ] `POST /api/subscription/webhook` validates the shared secret in the Authorization header.
- [ ] Returns 401 for invalid or missing secret.
- [ ] `INITIAL_PURCHASE` event sets parent and all linked children to `premium`.
- [ ] `EXPIRATION` event sets parent and all linked children to `free`.
- [ ] `RENEWAL` event updates `subscriptionExpiry`.
- [ ] `CANCELLATION` event does NOT immediately revoke premium (waits for expiry).
- [ ] Events for unknown `app_user_id` are logged and return 200 (no retry).
- [ ] All updates happen in a single Prisma transaction.

### AC-4: Free Tier Limits — API

- [ ] Free users can read at most 5 news articles per day (based on `news_viewed` ActivityLog count).
- [ ] Free users can view at most 5 reels per day.
- [ ] Free users get at most 3 quiz questions per session.
- [ ] Free users can only filter news/reels by their first `favoriteSport`.
- [ ] Premium users have no limits on any of the above.
- [ ] Parental limits still apply to premium users (parental guard runs first).
- [ ] 403 responses include `error`, `limitType`, `limit`, `used`, and `tier` fields.

### AC-5: Family Plan

- [ ] A parent's premium subscription propagates to up to 3 children linked via `parentUserId`.
- [ ] The 4th child linked to the same parent does NOT get premium (returns free tier limits).
- [ ] When a parent's subscription expires, all children revert to free.
- [ ] When a new child is linked to a premium parent, the child immediately gets premium (if under 3 children).

### AC-6: Mobile — Purchase Flow

- [ ] RevenueCat SDK initializes on app launch with the parent's user ID.
- [ ] Upgrade screen displays monthly and yearly pricing from RevenueCat offerings.
- [ ] Tapping "Monthly" or "Yearly" triggers the native purchase sheet.
- [ ] After successful purchase, user tier updates to premium (via webhook + status poll).
- [ ] Restore purchases recovers a previously purchased subscription.
- [ ] Anonymous users see "Create an account first" and are redirected to Register.
- [ ] Children see "Ask your parent" and are redirected to ParentalControl.

### AC-7: Mobile — Limit UX

- [ ] When a free user hits the news limit, `LimitReachedModal` appears with news-specific copy.
- [ ] When a free user hits the reels limit, `LimitReachedModal` appears with reels-specific copy.
- [ ] When a free user hits the quiz limit, `LimitReachedModal` appears with quiz-specific copy.
- [ ] Tapping a locked sport chip shows `LimitReachedModal` with sport-specific copy.
- [ ] "Upgrade" button in the modal navigates to Upgrade screen (for parents).
- [ ] "Ask your parent" button navigates to ParentalControl screen (for children).
- [ ] Usage counters ("3/5 articles today") are visible on HomeFeed, Reels, and Quiz for free users.

### AC-8: Web

- [ ] `/upgrade` page renders feature comparison and app store links.
- [ ] `LimitReached` component shows upgrade prompt for subscription-related 403 errors.
- [ ] Free usage counter appears on HomeFeed for free users.

### AC-9: i18n

- [ ] All subscription UI text uses `t()` with keys from `subscription.*` namespace.
- [ ] Both `es.json` and `en.json` have complete subscription key coverage.
- [ ] No hardcoded visible text in subscription-related components.

### AC-10: Security

- [ ] Webhook endpoint validates shared secret; rejects requests without it.
- [ ] Children cannot initiate purchases (UI prevents, API does not expose purchase routes).
- [ ] Subscription tier cannot be set via `PUT /api/users/:id` (only via webhook).
- [ ] Rate limiting applies to the webhook endpoint (default tier).
- [ ] No PII is sent to RevenueCat beyond the user ID.
