## Prerequisites

```bash
bash specs/mvp-to-product-2/create-environment.sh
```

## Validation Steps

### Feature 1: React Native Error Boundary

1. **Action**: Open the mobile app in Expo Go.
   **Expected**: The app loads normally without any error screen.

2. **Action**: In development mode, temporarily add `throw new Error('Test crash')` inside any screen component's render method (e.g., HomeFeed). Reload the app.
   **Expected**: A kid-friendly error screen appears with a stadium emoji, "Oh no! The game crashed!" title, a message, and a "Restart" button. In dev mode, a scrollable stack trace section appears below.

3. **Action**: Tap the "Restart" button on the error screen.
   **Expected**: The app recovers and re-mounts the component tree. Normal navigation resumes.

4. **Action**: Remove the test error and reload.
   **Expected**: App works normally.

### Feature 2: JWT Tokens in expo-secure-store

5. **Action**: Log in to the mobile app (or create a new account).
   **Expected**: Login succeeds normally. Tokens are now stored in SecureStore (not visible directly, but the app should work identically).

6. **Action**: Close and reopen the app.
   **Expected**: The session persists — the user remains logged in.

7. **Action**: Log out and log back in.
   **Expected**: Login/logout flow works correctly with the new secure storage layer.

### Feature 3: YouTube Embed Sandbox Parameters

8. **Action**: Navigate to a Reel that is a YouTube video on the **web app** (http://localhost:3000/reels).
   **Expected**: The video plays inline. No related videos appear at the end. No fullscreen button is visible. No annotations overlay the video.

9. **Action**: Right-click > Inspect the YouTube iframe in browser DevTools.
   **Expected**: The iframe has `sandbox="allow-scripts allow-same-origin allow-presentation"`. The `src` URL contains `rel=0`, `modestbranding=1`, `iv_load_policy=3`, `disablekb=1`, `fs=0`.

10. **Action**: Navigate to a YouTube Reel on the **mobile app**.
    **Expected**: The video plays normally. The embed URL should contain `rel=0`, `modestbranding=1`, `iv_load_policy=3`, `disablekb=1`, `playsinline=1` but NOT `fs=0`.

### Feature 4: Fail-Closed Content Moderation

11. **Action**: Trigger a manual news sync via `POST http://localhost:3001/api/news/sync` (use curl or Postman). Ensure the AI provider is running (Ollama by default).
    **Expected**: The sync completes. New articles are moderated and either `approved` or `rejected`.

12. **Action**: Stop the AI provider (e.g., stop Ollama). Set `NODE_ENV=production` and restart the API. Trigger another sync.
    **Expected**: New articles should have `safetyStatus: 'pending'` (not auto-approved). Check the database or the admin endpoint.

13. **Action**: Query the admin moderation endpoint: `GET http://localhost:3001/api/admin/moderation/pending` with an admin JWT.
    **Expected**: Returns 401 without auth. Returns 403 with a non-admin user. Returns 200 with pending articles for an admin user.

14. **Action**: Set `MODERATION_FAIL_OPEN=true` in the environment and restart. With AI still down, trigger a sync.
    **Expected**: Articles are auto-approved despite production mode (override is active).

15. **Action**: Check API logs after sync when there are stale pending articles (older than 30 minutes).
    **Expected**: A warning log message appears: "Stale pending articles detected — AI moderation may be down" with count and age details.
